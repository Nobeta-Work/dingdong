# 叮咚商城开发迭代说明

> [!abstract] 开发目标
> 2026-07-23 前交付可通过 Docker Compose 启动、可展示买家端与管理端页面的 `v0.9` 实训版。本文件描述普通开发流程，不预设固定角色。

## 1. 开发原则

- 先验证基础设施，再实现业务功能。
- 先确定数据模型和接口契约，再编写页面与服务代码。
- 优先保证登录、商品浏览、购物车、下单、模拟支付主路径。
- 每次提交保持小而完整，避免把多个无关功能混在一起。
- 07-22 起只修复缺陷，不再扩展新增业务范围。

## 3. 普通开发流程

1. 阅读 [[PRD|产品需求文档]] 与接口约定，明确本次迭代的验收条件。
2. 从 `main` 创建功能分支，先补充数据表或接口草案。
3. 完成服务端实现、单元/接口自测和必要的 OpenAPI 文档。
4. 完成前端页面或调用示例，并进行一次端到端联调。
5. 提交前执行 Maven 编译、相关测试和 `git diff --check`。
6. 合并后重新启动 Compose 与受影响服务，确认主路径没有回归。

## 4. 每项任务的完成标准

- [ ] 代码可编译，未提交 `target/`、本地配置或密码。
- [ ] 接口文档包含请求、响应和主要错误码。
- [ ] 正常、无权限、参数错误三类场景已自测。
- [ ] 数据库变更使用版本化 SQL 脚本。
- [ ] 页面或接口已完成一次前后端联调。
- [ ] 变更已补充 README、演示数据或运行说明（如适用）。

## 5. Git 与变更约定

- 分支命名、提交格式和合并方式遵循 [[GIT|Git 流程说明]]。
- 接口字段、枚举、状态码变化时，先更新文档再修改调用方。
- 卡住超过 30 分钟时记录现象、已尝试操作和报错，避免无记录重复试错。
- 影响订单、库存或支付的数据问题优先处理，并在合并前完成回归验证。

## 6. 当前实施状态（2026-07-17）

当前处于 **v0.5 后端主链路完成、v0.6 前端联调进行中** 阶段，尚未达到 v0.9 交付冻结条件。

| 区域 | 当前状态 | 依据与边界 |
|---|---|---|
| 基础设施 | 已完成 | Compose 已覆盖五个后端 Java 服务及 MySQL、Redis、Nacos、RocketMQ。 |
| 服务治理 | 已完成 | 五个服务默认启用 Nacos 服务发现；Gateway 通过配置类维护 `lb://服务名` 路由，由 Nacos 解析健康实例。 |
| 后端业务 | 已完成主路径 | 购物车已迁移至 Redis；用户取消订单、管理端商品/订单查询详情、经营统计、库存锁定、模拟支付、发货/收货、延时关单与 Outbox 重试均已有实现。 |
| 前端 | 已搭建，待联调 | 买家端与商品管理页面已存在；管理员订单和经营概览后端接口已具备，前端页面仍待接入。 |
